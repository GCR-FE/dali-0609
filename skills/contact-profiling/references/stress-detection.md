# 压力状态检测 · 五步判定闭环

> **用途**:识别客户当前的压力状态,**防止把压力下的临时行为误读为人格特征**。
>
> **为什么重要**:客户在压力下的行为会偏离基线,这种偏离是**高价值信号**(可能暴露核心恐惧、身份威胁、外部环境压力)。但如果直接把"偏离"当"压力溢出"报给 User,在中国商业文化下会大量误判("说抱歉"可能只是客套)。

## 五步判定闭环

### 步骤 1 · 压力源识别

区分压力来自哪里(不同压力源对应不同应对策略):

| 类型 | 来源 | 典型场景 |
|---|---|---|
| **外因压力** | 组织 / 业绩 / 合规 / 市场 | 裁员、业绩下滑、监管检查、竞品入场 |
| **内因压力** | 身份威胁 / 核心恐惧触发 | 被质疑专业度、方案暴露风险、个人决策被推翻 |
| **情境压力** | 临时场合 / 临时议题 | 有上级在场、在新人面前、议题敏感 |

> 信号来源:Phase 0 的结构化提问(公司处境、项目意义、他的职业阶段)+ 当下对话线索。

### 步骤 2 · 三通道信号提取

从本次互动中提取偏离信号:

| 通道 | 抓什么 |
|---|---|
| **语言通道** | 用词是否比基线更硬 / 更客气 / 更回避? |
| **语调通道** | 语速 / 音量 / 停顿是否与基线不一致? |
| **动作通道** | 姿势 / 眼神 / 动作节奏是否偏离? |

### 步骤 3 · 基线偏离判定

**这条必须做到**:对照**这个客户自己的基线**(存档 Profile 中记录),**不是人群通用基线**。

| 判定 | 条件 | 处理 |
|---|---|---|
| **非偏离** | 信号在该客户基线范围内,即使 User 感觉异常 | 不标记(可能是 User 的主观反应) |
| **单点偏离** | 只有一个通道偏离,其他两通道正常 | 记录"单点偏离 · 待观察" |
| **多点偏离** | 两个以上通道同时偏离,且一致性强 | 进入步骤 4 判级 |

**首次观察(没有基线)** → 只记录,不判定偏离,标注"暂无基线,待建立"。

### 步骤 4 · 压力等级

| 等级 | 条件 | 处理 |
|---|---|---|
| **轻** | 单点偏离,一次性 | 记录,标注"待观察" |
| **中** | 多点偏离,但有合理情境解释 | 提示 User:"这个反应可能跟 X 场景有关,要不要我们一起看一下?" |
| **重** | 多点偏离 + 累积出现 ≥2 次 + 无合理情境解释 | 主动标注 `[! 压力溢出]`,详细解读 + 具体应对 |

### 步骤 5 · 沟通应对

根据压力源类型 × 等级给出具体应对:

| 压力源 | 等级中 | 等级重 |
|---|---|---|
| **外因压力** | 用"我理解这个阶段你们面对的 X 压力"开场,展示同理心 | 临时调整方案,加入缓冲机制(分期 / 试点 / 降门槛) |
| **内因压力** | 避开触发点,改用迂回路径 | 直接承认"我可能刚才那段信息方式不对,重新来一下" |
| **情境压力** | 不触碰敏感议题,留到单独场景 | 主动重新约见("这件事我们找个独立时间再谈") |

## 压力导致的框架错位(核心注意事项)

压力下,客户的行为会让六框架判断出错:

### DiSC 错位

压力下一个 S 型的人可能临时表现出 D 型行为(急促、直接)。不要基于压力下一次观察就重判 DiSC 型。

### 九型型迁移

压力下每个九型会迁移到另一个型的**不健康状态**(见 `enneagram.md` 最后一节)。不要把迁移后的行为当新的型基线。

### 马斯洛层跳变

压力会让激活层瞬间降到安全层(2 层)。一个平时在尊重层(4 层)的人,在公司裁员消息传出后会瞬间变保守 — 这不是"他变了",是"他的激活层变了"。

### MBTI 下位功能

高压下,主导认知功能会让位给**下位功能**(inferior function) — 一个 Te 主导的人压力下可能变得情绪化、自怜(Fi 下位),这是临时状态,不是 MBTI 的重定位。

## 中国商业文化阈值校准

**基本思路**:西方行为分析默认的"情绪压抑 vs 表达"阈值,放在中国场景要往**情绪表达少的那一端**调整。

不要把常态客套行为直接套用西方临床心理学的"溢出 / 阈值"叙事:

| 行为 | 不一定是压力溢出 |
|---|---|
| "抱歉""辛苦""不好意思" | 基线中频繁使用的人,说这些不构成溢出 |
| "聊两句家常" | 基线中偶尔会聊的人,这次聊不构成信任升级 |
| "约咖啡厅" | 如果这个客户之前也约过非正式场景,不构成角色重定义 |
| "说慢了" | 口音重的人 / 天然慢语速的人,慢不是压力 |
| "提到个人事情" | 文化偏爱"拉近"的人(沿海 / 南方商业圈常见),聊私事是常态 |

**建立基线的最小样本**:
- 至少 2 次不同场景的观察
- 至少涵盖 3 个通道的基线(语言 / 语调 / 动作)
- 最好包含一次"正常状态"和一次"小压力"的对比

## 压力标记的输出格式

当判定为**压力等级重**时,输出里应该有如下标注(表达层,自然语言化):

```
注意:
他今天的状态跟之前 3 次见面不太一样 —
  语速比平时快了明显的一档
  打断了你 2 次(平时都会等你说完)
  答"我再看看"的时候没有像之前那样给后续问题

这可能跟 [具体事件 · 例如:他们公司上周公告的战略调整] 有关,
不一定是你方案本身的问题。建议:
1. 这次会议不要再加新议题,把该聊的闭环
2. 下次见面先空出 5 分钟让他讲讲公司近况
3. 如果他没主动聊,不要问 — 等他自己愿意说
```

**关键**:
- 不说"压力溢出""基线偏离""阈值"这些术语
- 用具体观察描述 + 可能的外部解释
- 给 User 的下一步动作必须具体到"做什么 / 不做什么"

---

## 不要做的判定

- ❌ 没有基线就直接判定"反常"
- ❌ 把单次偏离当作"压力溢出"
- ❌ 忽略外部环境直接归因为内因(他人格有问题)
- ❌ 把中国商业文化常态行为当西方临床心理学"溢出"
- ❌ 把压力下的 DiSC / 九型 / 马斯洛变化当成框架重判
- ❌ 对 User 使用"压力阈值""基线偏离"等术语 — 永远翻译成"今天跟之前不太一样"
