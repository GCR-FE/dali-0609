# 信任五框架 · 动态修正器

> **用途**:信任五框架是**贯穿修正器**,不在六框架级联中单独排位。它做一件事:**根据当前关系阶段修正对每个行为的解读**。
>
> **核心洞察**:同一个行为,在"供应商"关系阶段和"可信顾问"关系阶段意味着完全不同的事。不考虑关系阶段就直接读行为,会读错。

## 五个框架的分工

| 框架 | 回答什么 | 主要用途 |
|---|---|---|
| **1. Maister 信任方程式** | 什么构成信任? | 诊断缺哪块导致不信任 |
| **2. Lewicki-Bunker 三阶段** | 信任是怎么建立的? | 判断当前处在哪个阶段 |
| **3. 关系阶梯** | 销售关系的档位? | 判断当前档位和下一档 |
| **4. Speed of Trust** | 有哪些行为能快速建信任? | 指导具体销售动作 |
| **5. Maister 五步法** | 怎么把陌生客户带到信任关系? | 给销售流程蓝图 |

---

## 1. Maister 信任方程式

```
信任 = (可信度 + 可靠度 + 亲近度) / 自我导向
```

| 变量 | 定义 | 销售侧典型表现 | 客户侧识别 |
|---|---|---|---|
| **可信度(Credibility)** | 他相信你懂这个领域 | 你展示的专业知识、案例、数据 | 客户会问"你做过哪些类似的" |
| **可靠度(Reliability)** | 他相信你会做到你说的 | 你答应的事按时做到;不放鸽子 | 客户会记你之前承诺的事 |
| **亲近度(Intimacy)** | 他愿意跟你说真话 | 愿意分享非标准答案、内部情况 | 客户主动聊非业务话题 |
| **自我导向(Self-Orientation)** | 你关心自己的订单 vs 关心他的问题 | 说话时你是"我要卖"还是"你需要什么" | 客户能感觉到你是不是在 push |

**诊断应用**:
- 客户不回复邮件 → 可能可靠度没问题,但可信度/亲近度没建起来
- 客户答应但不执行 → 可能自我导向过高,他觉得你只关心订单
- 客户愿意见但不讲真话 → 亲近度低
- 客户愿意讲但不执行 → 可能可信度低(他觉得你方案不靠谱)

## 2. Lewicki-Bunker 三阶段

信任的建立是**有阶段性**的,不能跳阶段。

| 阶段 | 特征 | 典型场景 | 销售要做什么 |
|---|---|---|---|
| **计算型信任** | "我合作因为对我有利" | 刚认识、商业往来 | 清晰价值、明确条款、按承诺兑现 |
| **知识型信任** | "我信你因为我了解你会怎么做" | 合作过 2-3 次 | 持续一致性、可预测性、透明沟通 |
| **认同型信任** | "我信你因为我们价值观一致" | 深度合作多年、共同经历 | 展示深层价值观契合、共同愿景 |

**关键规则**:
- 计算型阶段硬推"认同感"会被客户觉得突兀
- 认同型阶段还用计算型话术会让客户觉得你没进步

## 3. 关系阶梯(销售特定)

| 档位 | 客户眼中的你 | 典型互动 |
|---|---|---|
| **供应商** | 可替换的卖东西的人 | 走招标流程;比价 |
| **可信资源** | 值得合作的一家 | 主动找你;愿意告知需求 |
| **问题解决者** | 能帮他解决具体问题的人 | 遇到问题先找你看看;愿意讨论非标方案 |
| **可信顾问** | 他决策时会参考你意见的人 | 愿意分享非公开信息;在内部替你讲话 |
| **战略伙伴** | 长期捆绑、深度绑定 | 你出现在他的长期战略里;风险共担 |

**识别档位的信号**:
- 见面场景变化(办公室 → 咖啡厅 → 家里吃饭)
- 话题范围(只业务 → 行业观察 → 个人话题)
- 信息透明度(官方说法 → 非标内情)
- 决策参与度(事后告知 → 事前征询)

## 4. Speed of Trust 行为清单(Covey)

13 个快速建信任的行为。销售场景常用的:

| 行为 | 怎么做 |
|---|---|
| **Talk Straight** | 直说你的看法,不绕弯 |
| **Demonstrate Respect** | 对客户员工、对手、同行都保持尊重 |
| **Create Transparency** | 主动说弱点、说局限、说不确定 |
| **Right Wrongs** | 做错了主动承认、补救 |
| **Show Loyalty** | 不在客户背后说客户坏话、不跟别的客户泄密 |
| **Deliver Results** | 说到做到 |
| **Get Better** | 持续学习,每次互动更有价值 |
| **Confront Reality** | 直面难题,不回避 |
| **Clarify Expectations** | 事前明确预期,不留后患 |
| **Practice Accountability** | 不甩锅,结果我负责 |
| **Listen First** | 先听再说,不抢话 |
| **Keep Commitments** | 承诺了就做到,做不到提前说 |
| **Extend Trust** | 先给对方信任,信任会回流 |

---

## 5. Maister 五步法

把陌生客户带到信任关系的销售流程:

1. **Engage** — 让客户愿意见你(进门票)
2. **Listen** — 真听他的问题(不是 pitch)
3. **Frame** — 把问题定义清楚(展示你懂)
4. **Envision** — 描绘解决后的样子(未来共识)
5. **Commit** — 明确下一步行动(闭环)

## 信任框架与其他框架的协作

- **信任是修正器,不是独立层**:同一个行为(例如"他约你去咖啡厅")在不同信任档位意味着不同
  - **供应商**档位:可能是例行拉近关系
  - **可信资源**档位:可能是角色重定义(想聊更深的)
  - **可信顾问**档位:可能是有不方便在办公室说的事
- **信任档位决定恐惧层可见度**:关系档位越深,他越敢让你看见真正的恐惧
- **信任档位决定商业/个人人格切换**:关系越深,他越愿意放下职位面具(见 SKILL.md 的角色松弛时刻)

## 销售场景的综合应用

```
当前信任档位(关系阶梯)
   ↓
决定哪些行为是"反常"(如果阶梯低,非正式场景就是异常)
   ↓
信任方程式诊断缺哪块(可信/可靠/亲近/自我导向)
   ↓
用 Speed of Trust 行为清单选 2-3 个具体动作
   ↓
符合 Lewicki-Bunker 当前阶段的沟通话术
   ↓
按 Maister 五步法推进到下一档
```

## 不要做的判定

- ❌ 跳档位判断("他愿意见你 = 可信顾问")— 必须多信号交叉
- ❌ 在早期档位就推深度话题(会被认为唐突)
- ❌ 对 User 使用"Maister""Speed of Trust""关系阶梯"等术语 — 永远说"你在他眼中是 X 角色"
- ❌ 把信任建立看成线性路径 — 可能倒退(重大事件能瞬间倒退档位)
