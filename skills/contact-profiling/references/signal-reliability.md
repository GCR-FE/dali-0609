## 为什么需要这一层

V1.6 把信号强度分成 strong / moderate / weak 三级,但实践中 Hermes 文档(Munkh 让另一个 AI 跑的)指出一个更深的问题:

> **不是所有信号都一样可信。客户在酒桌上拍胸脯说"这事包在我身上",和他在公司采购评审会上说"我推荐这个供应商",这两句话的可信度完全不一样。**

所以 V2.0 引入**五级可靠度评分(R1-R5)**,从信号产生的"场景压力 + 表演动机"两个维度切入,给每个信号一个更细的可靠度标签。

---

## 五级可靠度

### R5(最可信):无意识行为,无社交压力

**定义**:客户在没有"表演"动机的情况下自然流露出来的行为。

**为什么最可信**:他不是在说给你听,他甚至不知道你在观察。这种信号最真实,最不可能伪装。

**典型例子**:
- 他对下属说话的方式("把那个文件拿来"的语气) — 对下属没有表演必要
- 他在等电梯时不经意的表情 — 以为社交时间结束
- 他处理突发状况的反应(会议中接到紧急电话) — 压力下的真实反应
- 他车里 / 办公室个人区的状态 — 无意识的自我暴露

### R4(很可信):有压力的场景下的真实反应

**定义**:客户在有真实利益 / 情感压力、但不是在"讨好你"的场景中表现出来的行为。

**典型例子**:
- 谈判中突然提高嗓门:"这个价格不可能" — 真实的压力表达
- 内部会议上(你恰好在场)跟同事争论你的方案 — 在内部帮你说话
- 收到报价后第一反应是皱眉然后说"贵了" — 未经修饰的即时反应
- 项目出问题时发火 — 压力下的反应模式

**关键特征**:有真实的利益/情感压力存在,客户没时间"修饰"反应。

### R3(中等可信):正式商务互动中的表态

**定义**:在正常商务场景中,客户经过思考后表达的态度和观点。

**典型例子**:
- 正式会议上说"你们的方案我们评估过了,基本满足需求"
- 邮件里写"初步来看你们有竞争力,请安排下一步技术交流"
- 电话里说"我们最近确实在看这块,你发个材料我看看"

**为什么只是中等**:正式商务场景中,人会"管理"自己的表达。可能是真心,也可能是给余地继续谈价格。不确定但也不是没意义。

### R2(可信度低):社交场景中的表态

**定义**:在社交 / 半社交场景中,客户为了维系关系而说的话。

**典型例子**:
- 饭局上:"你们公司不错,有机会一定合作!"
- 行业会议偶遇:"改天约个时间好好聊聊!"
- 微信聊天:"好的好的,回头我看看"
- 朋友介绍认识后:"张总是个靠谱的人,以后多支持!"

**为什么可信度低**:中国商务文化里,这些话不代表客户在骗你 — 他可能当时真觉得"有机会合作" — 但他对每个供应商都这么说。

**关键判断标准**:**如果一句话拿掉也不影响当下的社交和谐,那它大概率是低可信度的。** "有机会合作"拿掉,换成"嗯嗯好的",场面也不会尴尬 — 所以这就是一句填充话。

### R1(最不可信):纯客套话

**定义**:纯粹出于礼貌和社交规范说的话,几乎没有任何信息量。

**典型例子**:
- "改天一起吃个饭啊" — 说完双方都知道不会真约
- "你们方案挺好的"(说完没有任何 follow up 动作)
- "我跟领导说说"(但你知道他根本不会提)
- "最近太忙了,等忙完这阵子……"
- "欢迎常来坐坐" — 等于"再见"

**R1 的处理方式**:**直接忽略**。不要把客套话录入 Record,它会污染画像的准确性。

## 中国场景修正规则

通用 R1-R5 框架放到中国 B2B 场景里,需要做几条本土化修正。

### 修正一:当面拒绝 → 可靠度 +1

中国文化里,当面说"不"心理成本很高。所以:

**如果客户当着你的面明确说了负面评价,这件事的可信度比同等级别上调一档。**

例:
- "你们这个报价比 XX 高了 20%,我没法交代" — 升级到 R4(原本 R3)
- "你们的服务响应我不太满意" — 升级到 R4

底层逻辑:愿意当面说的话,通常是深思熟虑过的真话。

### 修正二:当面夸赞 → 可靠度 -1

中国文化鼓励"给面子",所以:

**当面说的正面评价要自动降一档。**

例:
- "你们方案是我看过的最好的"(然后没有后续动作) — R3 降到 R2
- "我特别认可你们的理念"(没行动配合) — 降一级

判断标准:**正面评价 + 实际行动 = 可信** ; **正面评价 + 没有后续动作 = 客套**。

### 修正三:国企表态 → 可靠度 -1

国企决策文化的特殊性:
- 国企领导考虑合规风险,不可能个人说了算
- 讲究"集体决策",流程必走
- 说话更含蓄、更官方

例:
- 国企处长说"我个人觉得你们方案不错" → R3 降到 R2 ("个人觉得"是保护性措辞)
- 国企科长说"这个我要上会讨论" → 可能是真的,也可能是用流程推脱
- 国企领导非正式场合说"没问题" → 一定打折,通常只意味"我不反对",不意味会主动推进

### 修正四:民企老板 → 可靠度 ×1.5

民企老板(真正的实控人)的表态特别值钱:
- 他说了算,没人能否决
- 他时间宝贵,不应付不重要的人
- 他说"做"就真的会做

例:
- 民企老板在办公室说"这个事能帮到我们,你出方案明天发我" → R3 × 1.5 ≈ R4.5
- 民企老板饭局说"咱们合作一下" → R2 × 1.5 ≈ R3

**例外**:民企老板身边有政府/大客户在场时,他可能在"表演",这种情况不适用 ×1.5。

## 核心思维:同一个人 ≠ 同一个可信度

整套 R5-R1 体系最想教的一件事:

> **不要因为"这是张总说的"就觉得可信或不可信。关键是张总在什么场景下、出于什么动机、面对什么人说的。**

同一个张总:

| 场景 | 说的话 | 可靠度 |
|---|---|---|
| 对下属发火 | "这个供应商不行" | R5(非常可信) |
| 评审会上 | "这个供应商需要改进" | R4(比较可信) |
| 在你面前 | "你们还有些地方需要完善,但总体不错" | R3(中等) |
| 饭局上 | "合作嘛,都好说,有些小问题不影响" | R2(不太可信) |
| 初次见面寒暄 | "久闻贵公司大名" | R1(纯客套) |

五句话都是张总说的,可信度从 R5 到 R1 差 4 个等级。如果不区分,Record 就是一团浆糊。

## 信号叠加规则

### 低可靠度信号能叠加成高可靠度吗?

**不能。** 3 个 R2 还是 R2。一个人客套一次是客套,客套三次还是客套。

但是 — **如果不同场景、不同类型的信号都指向同一结论**,可以提升 Record 中该结论的 confidence:
- 饭局上说支持(R2) + 会后私下微信发内部材料(R4 行为) + 下属告诉你"张总开会提到你们了"(R3 转述)
- 三个不同类型 + 三个不同场景 → 结论的 confidence 升级

### 一个 R5 能否推翻多个 R2/R3?

**可以。** 你跟客户聊了 5 次,每次都很支持(5 个 R3)。但你看到他内部群发"A 供应商(竞品)综合评估最优"(R5 信号) — **一个 R5 推翻前 5 个 R3**。

因为他对你友善可能只是维系关系,而内部真实表态才是他的真实倾向。

## 可靠度降级规则(信号会过期)

跟 V1.6 时间衰减规则一致:

### 规则一:转述降级

亲眼看到的 vs 别人告诉你的,可信度天差地别。

- 你亲耳听到 — 原级
- 同事告诉你"客户好像说过..." — 降一级
- 客户的下属跟你说"我们领导觉得..." — 降两级

转述链越长,降级越多。"我听说他们那边有人提到过……" — 几乎可以忽略。

### 规则二:时间降级

| 数据时效 | 衰减规则 |
|---|---|
| 1 个月内 | 原级,直接使用 |
| 1-6 个月前 | × 0.8(略降) |
| 6-12 个月前 | × 0.5(降一档) |
| 1 年以上 | × 0.3(仅作参考) |
| 2 年以上 | × 0.1(几乎需要重建) |

为什么这么严:B2B 销售周期里,人的职位会变、生活会变、组织环境会变。30 天前的判断今天可能已经不准。

### 规则三:情境强度降级

借鉴 Hermes Part 7 的情境强度理论:

| 情境 | 性格 / 真实意愿信号的可靠度修正 |
|---|---|
| 极强情境(招标评审、合规审计、危机) | × 0.3 |
| 中情境(正式商务会议、项目汇报) | × 0.6 |
| 弱情境(饭局、私聊、非正式交流) | × 0.9 |

招标会上所有人看起来都"严肃认真",那不是性格,那是角色要求 — 这种场合下的信号要打折。

## 在 Record 里怎么用

如果纳入 V2.0 references 框架,每个 obs-* 观察可以加一个 reliability 字段:

```yaml
observations:
  - id: obs-001
    captured_at: "..."
    type: "verbal"
    content: "他对下属说'这个供应商不行'"
    context: "他以为我没听到,实际我在隔壁办公室"
    reliability: R5             # ← V2.0 新增字段
    reliability_basis: "无意识行为,他不知道我在听"
```

每个判断的证据链可以显示信号可靠度组合:

```yaml
disc_label:
  value: "D"
  evidence:
    - { obs: obs-001, reliability: R5 }
    - { obs: obs-002, reliability: R3 }
    - { obs: obs-003, reliability: R2, china_modifier: -1 }   # 当面夸赞降级到 R1
  confidence: medium  # 因为有一条 R5 撑腰,但其他偏低
```
