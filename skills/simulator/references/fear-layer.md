# 恐惧层 · 核心威胁认知的识别

> **用途**:恐惧层回答"他在保护什么"—— 是六框架级联中**最深的一层**,需要多次接触 + 多路信号交叉验证。
>
> **这几条必须做到**:
> - 首次接触 → **不允许**推到恐惧层
> - 2-3 次接触 → 可以记录**恐惧假设**,但必须标注"待验证"
> - 5 次以上 + 多路信号一致 → 才能作为相对可信的判断
> - 即便如此,对 User 输出时仍用"他对 X 非常敏感"这种自然语言,不说"核心恐惧"

## 恐惧层的来源与逻辑

恐惧层综合了:
- **九型核心恐惧**(恒定的内驱)
- **马斯洛受威胁时的退行层**(动态的外部激活)
- **职位身份威胁**(ToB 场景特有)

恐惧不是"负面情绪",是**驱动行为的底层威胁感**。每个人都有,但表现形式因人而异。

## 恐惧识别的三个关键窗口

### 窗口 1 · 防御性行为

当客户表现出不成比例的防御反应时,背后通常是被触碰到核心恐惧。

**典型防御行为**:
- 突然提高警惕、反问为什么要问
- 答非所问、转移话题
- 用"我们公司不是这样"否定假设
- 找理由提前结束会议

### 窗口 2 · 回避的话题

注意他**不说什么**,往往比他说什么更有信息。

**常见回避信号**:
- 你提到某个话题,他快速 skip 过去
- 你问他近期某件事,他只给官方说法
- 同一话题重复出现时他每次都用类似的套话

### 窗口 3 · 压力触发点

观察什么场景会让他行为偏离基线(见 `stress-detection.md`)。触发点通常指向恐惧。

**典型触发**:
- 被质疑专业度 → 可能触发 5 号"恐惧无能"
- 董事会 / 上级批评 → 可能触发 3 号"恐惧失败"
- 被迫做快决定 → 可能触发 6 号"恐惧孤立"或 1 号"恐惧出错"
- 被限制选项 → 可能触发 7 号"恐惧被困"
- 被下属挑战 → 可能触发 8 号"恐惧被控制"

## ToB 常见职位的恐惧原型

| 职位 | 常见核心恐惧 | 识别信号 |
|---|---|---|
| **CFO** | 数据错了上报、监管处罚、财务失控 | 反复核对口径;追问"谁签过字" |
| **CTO** | 技术决策被证明失败、系统事故、被换掉 | 强调"可控""可回退";关心 POC |
| **CEO**(一代) | 公司走下坡路、被行业甩开、个人声誉受损 | 关心行业位置、同行对比、战略叙事 |
| **COO** | 执行出事、流程崩溃、被指责 | 关心实施细节、风险预案 |
| **CIO**(转型期) | 数字化转型失败、IT 预算被削、被业务边缘化 | 关心业务绑定、展示成功案例 |
| **CHRO** | 人才流失、组织文化崩坏、被认为只做行政 | 关心变革影响员工情绪、长期建设性 |
| **采购 VP** | 供应商选错背锅、被审计质疑 | 关心流程合规、多供应商备份 |
| **新上任者**(任意职位) | 前三个月不出成果、被认为不合适 | 急于要快速 win、避免大动作 |

> **重要提醒**:以上是**先验分布**,不是标签。具体某人的恐惧要以观察为准。

## 恐惧层的销售应对

**做法**:销售不能去"挑战"恐惧(会让客户防御更强),要**帮客户管理他的恐惧**。

| 恐惧类型 | 销售要做什么 |
|---|---|
| **执行失败类** | 准备详细实施路径、参考案例、分阶段验证机制 |
| **专业权威类** | 不直接纠正,让他"自己发现";给他对上汇报的素材 |
| **合规监管类** | 主动提供合规文件、第三方认证、审计支持 |
| **被动失控类** | 给他决策权,不逼他快速 commit;保留回退机制 |
| **关系孤立类** | 找他信任的人做引荐;准备"同行也这么做"的背书 |
| **声誉行业类** | 帮他打造"行业领先""首创"的叙事 |

## 恐惧层与其他框架的协作

- **恐惧层 = 九型恐惧 × 马斯洛激活层 × 身份威胁**:同一个九型 6 号,在安全层被激活时(公司业绩下滑),恐惧会急剧放大
- **Dilts 身份层是恐惧的镜像**:他把自己定义为 X(Dilts 身份),当这个 X 被威胁时就触发恐惧
- **压力信号是恐惧的外显**:见 `stress-detection.md`,压力信号的根源往往是恐惧被触发

## 不要做的判定

- ❌ 基于单次场景就判定核心恐惧
- ❌ 对 User 说"他的核心恐惧是 X" — 永远翻译成"他对 X 非常敏感"
- ❌ 把恐惧当"弱点"去利用 — 销售策略应该是"帮他管理恐惧",不是"攻击恐惧"
- ❌ 忽略职位身份带来的先验恐惧(新上任者 / 转型期 CIO 等)
