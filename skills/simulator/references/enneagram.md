# 九型人格 · 底层动机与 ToB 销售场景的行为识别

> **用途**:九型人格回答"他为什么这么做"—— 是六框架级联中的第四层,直达底层动机结构。
>
> **使用原则**:
> - 至少需要 3 次接触 + 多路信号交叉才能初判九型
> - 九型是**恒定内驱力**,不随环境变化(这跟马斯洛"当前需求"不同)
> - 九型判断绝不对 User 暴露 — 内部用"他在追求 X / 回避 Y"表达

## 九型速查:核心恐惧与核心欲望

| 型 | 代号 | 核心恐惧(他回避什么) | 核心欲望(他追求什么) | 典型自我定义 |
|---|---|---|---|---|
| **1 号** | 完美型 | 被评价为堕落 / 错误 / 有缺陷 | 做"正确的事",保持正直 | "我是一个坚持正确的人" |
| **2 号** | 助人型 | 不被需要 / 不被爱 | 被需要、被感激 | "我是一个关心他人的人" |
| **3 号** | 成就型 | 失败 / 没价值 / 不被认可 | 成功、被赞赏 | "我是一个做成事的人" |
| **4 号** | 自我型 | 没有身份 / 平庸 | 独特性、真实表达 | "我是一个与众不同的人" |
| **5 号** | 理智型 | 无知 / 无能 / 被侵扰 | 有能力、有知识 | "我是一个懂的人" |
| **6 号** | 忠诚型 | 失去支持 / 没有指引 | 安全感、有所依靠 | "我是一个负责任的人" |
| **7 号** | 热情型 | 被困 / 无聊 / 痛苦 | 新鲜体验、保持选项开放 | "我是一个热爱生活的人" |
| **8 号** | 领袖型 | 被控制 / 被伤害 / 脆弱 | 掌控、自主 | "我是一个不被欺负的人" |
| **9 号** | 和平型 | 冲突 / 分裂 / 失去内在和谐 | 和谐、内在宁静 | "我是一个不起冲突的人" |

## ToB 销售场景常见型的识别信号

### CFO / 财务负责人常见:1 号 / 5 号 / 6 号

**1 号 · 完美型 CFO**:
- 关注合规、流程、数据准确性
- 听到不规范时会皱眉或直接指出
- 会问"这符合 XX 准则吗"
- 回避:被发现数据错误、被监管批评

**5 号 · 理智型 CFO**:
- 深度研究问题,不急下结论
- 提问极其精准,直指核心
- 不喜欢被打扰,需要独处思考时间
- 回避:被认为不懂 / 知识外露不够

**6 号 · 忠诚型 CFO**:
- 反复问"最坏情况怎么办"
- 喜欢"别人也用了吗"这种背书
- 决策前一定会找他信任的人再核一遍
- 回避:失去组织支持、孤立无援

### CEO / 一代企业家常见:3 号 / 7 号 / 8 号

**3 号 · 成就型 CEO**:
- 关注业绩、排名、成功案例
- 喜欢展示自己的成就
- 节奏快,要结果
- 回避:被看作失败者

**7 号 · 热情型 CEO**:
- 兴奋点多,话题跳跃
- 喜欢新概念、新机会
- 不喜欢被限制选项
- 回避:被固定在某个承诺里

**8 号 · 领袖型 CEO**:
- 强势、直接、要掌控
- 不容易被 push,但会主动 push 别人
- 尊重同样强势的人
- 回避:被控制、显示弱点

### 技术负责人常见:5 号 / 1 号

**5 号 · 理智型 CTO**:见上
**1 号 · 完美型 CTO**:关注架构规范、代码质量、技术债

### HR / 运营常见:2 号 / 9 号

**2 号 · 助人型 HR**:
- 关心团队感受
- 主动帮忙、承担额外工作
- 容易接受销售的"关心"
- 回避:被视为冷漠

**9 号 · 和平型运营**:
- 不表达反对意见
- 和稀泥、愿意听各方
- 慢决策(或不决策)
- 回避:内部冲突

## 识别信号:从行为推动机

| 观察到 | 可能的核心动机 |
|---|---|
| 一再追问风险、要 B 计划 | 6 号(恐惧失去支持)或 1 号(恐惧出错) |
| 主动展示成就、要曝光 | 3 号(恐惧没价值) |
| 拒绝被限制、要保留选项 | 7 号(恐惧被困) |
| 强硬、反击 push | 8 号(恐惧被控制) |
| 回避冲突、不表态 | 9 号(恐惧冲突) |
| 关心员工影响 | 2 号(恐惧不被需要)或 9 号 |
| 深度独处思考 | 5 号(恐惧无能 / 被侵扰) |
| 高标准、批评性 | 1 号(恐惧堕落) |
| 强调独特、不愿模仿 | 4 号(恐惧平庸) |

## 压力下的型迁移(九型理论)

压力下,每个型会迁移到另一个型的**不健康状态**:
- 1 → 4 压力下变情绪化、自怜
- 2 → 8 压力下变支配、愤怒
- 3 → 9 压力下变麻木、无动力
- 5 → 7 压力下变散漫、逃避
- 6 → 3 压力下变焦虑表现主义
- 7 → 1 压力下变挑剔、严苛
- 8 → 5 压力下变退缩、孤立
- 9 → 6 压力下变焦虑、担忧

> **应用**:当 User 描述的行为跟你建立的基线不一致,不要立刻推翻基线 — 先想想"是不是压力下的型迁移"。配合 `stress-detection.md` 使用。

## 不要做的判定

- ❌ 基于单次观察就定型 — 九型必须跨场景、多路信号验证
- ❌ 把九型当道德评判(没有"好型"或"坏型")
- ❌ 对 User 说"他是 3 号 / 8 号" — 永远翻译成"他在追求 X / 回避 Y"
- ❌ 把压力下的型迁移误读为"他变了一个人"
