# Dilts 六层 · 从行为向上推导信念、身份、愿景

> **用途**:Dilts 是**贯穿工具**,不在六框架级联中单独排位。它做两件事:
> 1. 识别客户在哪个层次说话,判断销售应该在哪个层次回应
> 2. 从可观察的行为向上推导到信念 / 身份 / 愿景
>
> **在 SKILL.md 的 Agent Persona 里**,"Dilts 教练沟通界面"是 Agent 与 User 互动的表达风格 —— 这里的 Dilts 指的是**推理工具**本身。两个用法不冲突。

## Dilts 六个逻辑层次

从下到上,每一层比下一层更难改变,但对行为的影响更大:

| 层级 | 名称 | 回答什么问题 | ToB 场景典型信号 |
|---|---|---|---|
| **6** | 愿景 / Spiritual | 我为谁?我的更大使命是什么? | "我希望我们这代人做的事能..."  |
| **5** | 身份 / Identity | 我是谁? | "作为 CFO,我..." / "我这种风格的人..." |
| **4** | 信念 / Beliefs | 什么是真的?什么是重要的? | "我相信 ROI 必须可追溯" / "我觉得人不能将就" |
| **3** | 能力 / Capabilities | 我怎么做? | "我们是怎么做这个的" / "我的强项是..." |
| **2** | 行为 / Behaviors | 我做了什么? | "昨天我跟 X 开了会" / "我批了这个预算" |
| **1** | 环境 / Environment | 在哪?和谁?什么时候? | "在我们公司" / "这个行业" / "2026 这个节点" |

## 说话要说到他在听的那一层

**销售常见错误**:客户在第 5 层(身份)表达,销售在第 2 层(行为)回应 —— 这叫"层次错配",客户会觉得"你没懂我"。

**正确做法**:识别他在哪层说话,**在同一层或稍高一层回应**。

### 示例

客户说:"作为一个对数字负责的人,我不能把这个数据拿到董事会去" → 这是**第 5 层(身份)**表达

| 销售回应 | 在哪层 | 效果 |
|---|---|---|
| "我们可以改一下报表格式" | 第 2 层(行为) | 层次错配,客户觉得没懂 |
| "我们的数据精度是可以保证的" | 第 3 层(能力) | 层次错配 |
| "我理解,对数字负责是你一直坚持的原则 — 我们一起看看怎么让这份报告经得起董事会的审视" | 第 5 层(身份) | 层次匹配,真正共情 |

## 从行为向上推:推导路径

**规则**:至少需要 2-3 条独立的、跨场景的行为观察,才能向上推一层。**跳层推导风险极高**。

### 推导示例

**观察行为**(层 2):
- 第 1 次见面:他在数据页停留最长,提出 3 个精度问题
- 第 2 次见面:他带着一份打印的数据,圈出 2 处认为不一致的地方
- 第 3 次见面:他在闲聊时提到"我做财务二十年了,没有因为数据被老板骂过"

**向上推能力**(层 3):
→ 他有很强的数据追溯和核对能力

**向上推信念**(层 4):
→ "数据必须可追溯才有意义" / "专业人士不能容忍数据错误"

**向上推身份**(层 5):
→ 他把自己定义为"一个对数字绝对负责的人"
→ 证据链:行为一致性 + 他自己的陈述 + 职业经历

**向上推愿景**(层 6 — 只有 5 次以上接触 + 主动表达才能推):
→ 可能是"留下一个无懈可击的财务体系"这类长期意义
→ 但 5 次以下必须留空或标"待验证"

## 身份层的特殊重要性

在 ToB 销售里,**身份层**(第 5 层)是最有销售杠杆的一层:

- 比愿景层(6)更具体可抓
- 比信念层(4)更稳定、更个人化
- 是"核心恐惧"的镜像 —— 他害怕失去的,往往正是他的身份定义

**识别身份层的三个信号**:

1. 他主动用 "作为一个 X 的人" 句式(最直接)
2. 他反复维护某种自我形象(即使场景不同)
3. 他的价值判断跟他的自我陈述一致(信念-身份共振)

**销售应用**:
- 让他在跟你的互动中感觉"我的身份被看到了" → 建立深层信任
- 避免触发身份威胁(见 `fear-layer.md`)
- 帮他在组织中强化他的身份(准备对上/对外的材料)

## 愿景层(Spiritual)· 高阶但少见

愿景层的表达通常只在:
- 职业后期 / 临近退休
- 二次创业 / 转型期
- 被信任到"可以说心里话"的关系阶段

**信号**:
- "我希望 X 年后回头看..."
- "这个行业应该变成..."
- "我们这代人能做的是..."

如果客户从未在这个层表达过,**不要硬推** —— 要么关系还没到,要么他本来就不是愿景型的人。

## Dilts 与其他框架的协作

- **Dilts 身份层 ↔ 九型核心自我定义**:互为佐证。如果九型判断是 3 号,Dilts 身份层陈述应该跟"成就""做成事"有共振
- **Dilts 信念层 ↔ 九型核心欲望/恐惧的价值化**:他"相信"什么,往往是他"想要"和"害怕"的道德包装
- **Dilts 环境层 ↔ 马斯洛当前激活层**:环境决定哪层马斯洛需求被激活

## 不要做的判定

- ❌ 跳层推导(从行为直接推愿景)
- ❌ 基于单次陈述就判定身份层 — 需要跨场景一致性
- ❌ 对 User 使用"Dilts""逻辑层次""身份层"这些术语
- ❌ 在关系早期强推愿景层(客户会觉得唐突)
- ❌ 把 Dilts 当成"阶梯" — 它是层次,不是高低优劣
