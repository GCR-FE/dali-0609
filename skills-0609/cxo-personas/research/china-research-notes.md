# 中国版 CXO Personas 研究笔记

## 一、中国企业分类框架（B2B 销售视角）

### 框架来源与依据

本分类框架综合以下权威来源：

| 来源 | 贡献 | 引用 |
|------|------|------|
| McKinsey | 三维模型：所有制×数字化成熟度×行业/增长 | McKinsey China B2B Market Sizing 2024 |
| BCG | 2×2矩阵：战略自主性×数字化野心 | BCG Digital Acceleration Index China 2024 |
| Bain | 五类买家分层（央企/地方国企/大型民企/SME/外企） | Bain China Technology Market Report 2024 |
| Roland Berger | 专精特新=独立买家类型（类比德国Mittelstand） | Roland Berger China Mittelstand Report 2023 |
| 国家统计局 | 大中小微四档划分标准（工信部联企业[2011]300号） | 16个行业各有营收+人数门槛 |
| SASAC/国资委 | 所有制分类：央企(~97家)/地方国企/混合所有制 | 国资委企业名录 |
| 工信部 | 专精特新三级体系：创新型SME→专精特新→小巨人 | 工信部专精特新认定标准 |
| 信创政策 | 强制替代分层：Tier1党政/Tier2重点国企/Tier3自愿 | 国家信创工程规划 |
| 华为铁三角 | S/A/B/C客户分级（营收潜力×战略价值×关系深度） | 华为大客户销售方法论 |
| 用友 | 按规模分产品线（大型→YonBIP/中型→U9/小型→畅捷通） | 用友产品矩阵 |

### 核心分类（6类）

**设计原则：** 每个类型必须在 CXO 决策行为上与其他类型有显著、可量化的差异。如果两类企业的 CXO 决策链、预算周期、供应商偏好无显著差异，则合并。

| # | 企业类型 | 英文标签 | 典型代表 | 独立理由 |
|---|---------|---------|----------|----------|
| 1 | **央企** | Central SOE | 中石油、国家电网、中国移动、中国银行 | 中组部任命、党委三重一大、集采制度、信创Tier1-2强制 |
| 2 | **地方国企** | Local SOE | 上汽集团、首钢、粤海集团、北京金控 | 省市国资委管、更灵活、政策执行滞后1-2年、但仍有党委前置 |
| 3 | **大型民企（含上市）** | Large Private | 字节跳动、比亚迪、美的、宁德时代、理想汽车 | 创始人驱动、CXO title西方化、资本市场/董事会压力、决策快 |
| 4 | **专精特新/中小民企** | SME / Little Giant | 12000+家工信部认证小巨人（营收1-4亿） | 创始人一言堂、预算小但决策极快、技术合作意愿强、无CXO title |
| 5 | **外企在华** | MNC China Sub | 微软中国、西门子中国、SAP中国、宝马中国 | 双线汇报、全球标准vs本地适配、采购权限61%已本地化 |
| 6 | **出海企业** | Going-Global | Anker、Shein、TikTok、蔚来海外、传音 | 双栈架构、多法域合规、SaaS支出同行2.1倍、多云67% |

| 未独立的类型 | 归入 | 理由 |
|-------------|------|------|

## 二、中国企业高管Title体系 vs 西方CXO

### 核心Title对照表

| 中文Title | 西方对应 | 备注 |
|-----------|----------|------|
| 董事长 | Chairman | 国企里常兼党委书记，是真正的最高权力 |
| 总裁 | President | 集团级，比总经理高；华为的"轮值董事长"是独特模式 |
| 首席执行官 | CEO | 非法定职务，主要科技/互联网/境外上市公司使用 |
| 总经理 | GM/CEO(视语境) | 公司法规定的最高运营职务，所有公司必须设置 |
| 副总裁 | EVP/SVP | 比西方VP更高级，更接近EVP |
| 副总经理 | Deputy GM/VP | 国企和传统企业的标准高管title |
| 总监 | Director/Sr. Director | 如市场总监、技术总监 |
| 首席XX官 | CXO | 主要科技/互联网公司采用 |

### 关键区别
- **总经理**是法定角色，**CEO**不是 —— 法律上只认总经理
- **总裁 vs CEO vs 总经理**：总裁≈集团President，CEO是品牌化称呼，总经理是法定运营一把手
- 中国公司更多用"XX负责人"、"分管副总"这种功能性描述，而不是CXO

### 各CXO角色在中国的普及度（按企业类型）

| 角色 | 央企/国企 | 大型民企 | 专精特新 | 外企在华 | 出海企业 |
|------|-----------|----------|----------|----------|----------|
| CEO | ✗ 用"总经理" | ✓ 高 | ✗ 用"总经理" | ✓ 高(双语) | ✓ 高 |
| CFO | ✗ 用"总会计师" | ✓ 高(上市) | ✗ 用"财务经理" | ✓ 高 | ✓ 高 |
| CTO | ✗ 用"总工程师" | ✓ 高(科技) | ✗ 用"技术总监" | ✓ 高 | ✓ 高 |
| COO | ✗ 无 | ✓ 中高(互联网) | ✗ 无 | ✓ 中 | ✓ 中 |
| CMO | ✗ 无 | ✓ 中(消费品) | ✗ 无 | ✓ 中 | ✓ 中 |
| CIO | ✗ 用"信息化负责人" | ✓ 中(大型) | ✗ 无 | ✓ 高 | ✓ 中 |
| CHRO | ✗ 用"人力资源部长" | ✓ 中(大型科技) | ✗ 无 | ✓ 高 | ✓ 中 |
| CDO | ★ 政策要求设置(2021+) | ✓ 低→中 | ✗ 无 | ✓ 中 | ✓ 低 |
| CAIO | ✗ 极少 | ✗ 用"AI负责人" | ✗ 无 | ✓ 低 | ✗ 用"AI VP" |
| CISO | ★ 数据安全法推动中 | ✓ 低→中 | ✗ 无 | ✓ 高 | ✓ 中 |

✓ = 普遍使用该title  ✗ = 使用替代title  ★ = 政策驱动中

## 三、按企业类型的组织架构

### 央企/国企
```
党委书记/董事长（常同一人）
├── 总经理/党委副书记
│   ├── 副总经理 × 3-6人（分管不同板块）
│   ├── 总会计师（法定，替代CFO）
│   ├── 总工程师（工业国企特有）
│   ├── 总经济师（部分国企）
│   └── 总法律顾问
├── 纪委书记
└── 工会主席
```
- 不用"首席XX官"，标配是副总经理体系
- 总会计师（替代CFO）是国务院法规要求
- 总工程师是工业国企特有的技术最高权威
- 领导任命来自中组部（央企）或省市组织部（地方国企）

### 大型民企（科技/互联网）
```
董事长/CEO（常为创始人）
├── 总裁/COO
├── CFO
├── CTO
├── 事业群总裁 × N
├── SVP/VP × N
└── 总监层
```
- 大量采用西方CXO title
- 扁平化、矩阵管理
- 特殊结构：阿里"合伙人制"、华为"轮值制"、字节"虚拟组织"

### 外企在华
```
大中华区总裁/CEO
├── 各职能VP（销售/市场/HR/财务）
├── 各业务线总经理
├── 政府关系/公共事务总监
└── 法务/合规总监
```
- 双语title标配
- 双线汇报（本地+亚太/全球）
- 重大决策需全球总部审批（但61%已本地化IT采购权）
- "法定代表人"是中国特有的法律要求

### 专精特新（中小隐形冠军）
```
董事长/总经理（同一人=创始人）
├── 副总经理 × 1-3
├── 技术总监/研发副总
├── 财务经理
└── 生产副总/厂长
```
- 高度创始人中心化
- 几乎不用CXO title
- 技术带头人虽无C-title但影响力巨大
- 任何¥10-50万+的采购创始人亲自看

### 出海企业
```
CEO/创始人
├── 国内业务线（CTO/COO/CMO）
├── 海外业务线（VP International / 出海VP）
│   ├── 各区域GM
│   ├── 海外合规/法务
│   └── 海外技术架构
├── CFO（统管两条线财务）
└── 平台/中台（数据、AI、供应链）
```
- 双栈IT架构（国内阿里云/华为云 + 海外AWS/GCP）
- 海外团队独立采购权
- 技术选型偏国际品牌（为了海外合规和人才招聘）

## 四、党委在国企决策中的角色

### "三重一大"制度
党委会必须前置审议：
1. 重大决策
2. 重要人事任免
3. 重大项目安排
4. 大额资金运用

### 决策流程
```
党委会（前置研究讨论）→ 董事会（正式决议）→ 总经理办公会（执行部署）→ 执行
```

### 对B2B销售的实际影响
- 重大合同（通常¥500万-5000万+）需过党委会
- 战略合作/并购：必须党委批准
- 中层以上人事：党委最终决定
- IT/数字化大项目：需党委签字
- **党委前置额外增加2-4个月决策时间**

### 央企 vs 地方国企差异

| 维度 | 央企 | 地方国企 |
|------|------|----------|
| 领导任命 | 中组部 | 省市组织部 |
| 监管方 | 国务院国资委 | 地方国资委 |
| 党委权力 | 极强，一切重大事项必过 | 强，但灵活度稍高 |
| 信创执行 | Tier1-2，严格时间表 | 跟随央企，滞后1-2年 |
| 采购门槛 | >100万公开招标 | 各地不同，200-500万 |
| 政治站位 | 极高，"国之重器" | 中-高，服务地方经济 |
| 对外合作灵活度 | 低（需层层审批） | 中（地方有自主权） |

## 六、中国CXO的独特优先级

### 1. 政策驱动议程（vs 西方市场驱动）

| 政策主题 | 影响的CXO | 实际意义 | 企业类型影响度 |
|----------|-----------|----------|---------------|
| 新质生产力 | 全体 | 所有投资必须能对齐此叙事才能获得补贴和政治支持 | 央企★★★★★ 民企★★★☆☆ |
| 数字中国 | CDO/CIO | 数据资产入表（2024.1生效）、数据作为生产要素 | 央企★★★★★ 民企★★★☆☆ |
| 十四五收官（2025）| CIO/CTO | 政府关联企业面临KPI截止日，冲刺完成数字化指标 | 央企★★★★★ 地方国企★★★★☆ |
| AI+制造 | COO/CTO | 智能工厂认证、预测性维护、机器视觉 | 央企★★★★☆ 民企★★★★☆ |
| 东数西算 | CTO/CIO | 数据中心选址须对齐国家算力枢纽布局 | 央企★★★★★ 大型民企★★★☆☆ |
| 双碳目标 | CFO/CSO | 碳核算强制要求，ESG是合规不是自愿 | 央企★★★★★ 出海★★★★☆ |
| 信创替代 | CIO/CTO/CISO | 核心系统国产替代时间表 | 央企★★★★★ 外企★★☆☆☆ |

**核心差异：政策驱动占中国CXO议程的30-50%（央企/国企），15-25%（大型民企），<5%（专精特新/出海企业）。西方企业<10%。**

### 2. 信创/国产替代（按企业类型分层）

| 企业类型 | 信创紧迫度 | 典型要求 | 对B2B销售的意义 |
|----------|-----------|----------|----------------|
| 央企(Tier1-2) | ★★★★★ | 2025-2027核心系统全替 | 必须有国产替代方案或信创认证 |
| 地方国企 | ★★★★☆ | 跟随央企，滞后1-2年 | 信创是加分项但非硬性 |
| 大型民企 | ★★☆☆☆ | 自主选择，部分金融受影响 | 不是首要考量，性能优先 |
| 专精特新 | ★☆☆☆☆ | 基本无要求 | 完全不关心，只看性价比 |
| 外企在华 | ★★☆☆☆ | 部分行业（金融、电信）受限 | 可能被迫用国产组件 |
| 出海企业 | ★☆☆☆☆ | 无要求 | 反而偏好国际品牌 |

信创栈：OS(麒麟/统信) → DB(达梦/OceanBase/TiDB) → 中间件(东方通) → 办公(WPS) → ERP(用友/金蝶)

2024-2025：金融核心系统进入信创"深水区"

### 3. 数据合规三法（数据安全法+个保法+网安法）

| 维度 | 对不同企业类型的影响 |
|------|---------------------|
| 数据出境 | 出海企业★★★★★（核心痛点）、外企在华★★★★☆、央企★★★☆☆ |
| 等保2.0 | 央企★★★★★（强制）、大型民企★★★★☆、专精特新★★☆☆☆ |
| 个人信息保护 | 所有类型★★★★☆（但执法力度不同） |
| CDO/DPO设置 | 央企★★★★★（政策要求）、大型民企★★★☆☆（自主） |

- CDO/DPO成为强制角色（央企/国企）
- 跨境数据流动受限 → 架构必须数据本地化
- 等保2.0比西方框架更具体/强制
- CEO有"第一责任人"法律概念 → 个人刑事责任可能

### 4. 出海战略（按企业类型的不同诉求）

## 十、中国CEO优先级（按企业类型，2024-2025）

### 央企CEO（一把手）优先级与考核

**SASAC 2024考核框架："一增一稳四提升"**

| 维度 | 指标 | 说明 |
|------|------|------|
| 一增 | 利润总额增长 | 盈利能力 |
| 一稳 | 资产负债率稳定 | 风险控制 |
| 四提升 | 净资产收益率 | ROE |
| | 研发投入强度 | 创新投入 |
| | 经营现金流比率 | 现金质量 |
| | 全员劳动生产率 | 效率 |

**2025新增：张玉卓"五个价值"（Five Values）**
- 经济价值（传统财务指标）
- 战略价值（国家战略对齐度）
- 安全价值（供应链/能源/信息安全）
- 创新价值（科技自立自强）
- 社会价值（就业、ESG、共同富裕）

**央企CEO实际优先级排序：**
1. 政治站位与党建考核（一票否决项）
2. 新质生产力落地（2024-2025核心叙事）
3. 信创替代时间表（核心系统2027前完成）
4. 国际化/一带一路项目
5. 数字化转型KPI（十四五收官冲刺）
6. ESG/双碳合规
7. 风险防控（反腐、合规、安全生产）

> **Sales implication:** 卖给央企CEO，必须对齐"新质生产力"叙事。任何方案如果不能用"新质生产力"语言包装，过不了第一关。

| CEO | 公司 | 公开stated优先级 | 来源 |

### 专精特新/SME CEO优先级

**数据画像（工信部2024）：**
- 平均研发强度：7-10%（远超大企业3-4%）
- 平均专利数：50+件/企业
- 细分市场占有率：70%+（小巨人认定标准）
- 平均营收：1-4亿元

**CEO实际优先级：**

4. 大客户关系维护（往往依赖1-3个核心客户）
5. 现金流管理（被大客户压款是常态）
6. 人才招聘（三四线城市招技术人才极难）

> **Sales implication:** 专精特新CEO最关心的是"帮我在细分领域保持技术领先"。任何能直接提升其核心技术能力的方案，创始人会亲自拍板。

**典型IT项目采购时间线（央企）：**


### 预算周期与Q4效应

| 数据点 | 来源 |
|--------|------|
| Q4占年度IT支出35-40% | IDC China/行业共识 |
| 12月单月可占年度支出15-20% | 财政部年度执行报告模式 |

- 11-12月：项目验收高峰 → 触发新一轮采购
- 政府/国企财政年度：严格日历年制
- **对销售的意义：** 9-10月是最佳pitch时机（赶Q4执行）

### 华为铁三角销售方法论

| 角色 | 英文 | 职责 |
|------|------|------|
| 客户经理(AR) | Account Representative | 客户关系、商务、合同 |
| 解决方案经理(SR) | Solution Representative | 技术方案、POC、交付设计 |

2. 普遍客户关系（影响层：处长/科长/项目经理）
3. 组织客户关系（战略层：长期合作框架、联合创新）

### 中国式销售最佳实践

**标杆参访（Reference Visit）— 中国B2B第一转化工具：**
- 安排潜在客户CEO参观已成交客户的部署现场
- 效果远超任何PPT或demo
- 核心：让"同行"（尤其是同行业/同规模）现身说法
- 参访后配合商务宴请深化关系

**关系建设节奏：**
```
中间人引荐 → 初次见面(短会/行业活动) → 商务宴请(加深了解) → 
标杆参访(建立信任) → 方案汇报(正式推进) → 高层互访(战略背书)
```

**宴请文化要点：**
- 饭局是关系深化的核心场景，不是成交场景
- 绝对不要在饭局上"逼单"
- 座位安排有严格讲究（主宾位、陪客位）
- 敬酒文化仍然存在（虽在减弱）
- 饭局的目的是从"供应商"变成"朋友"

### Discovery Questions：中国文化适配

**与西方的核心差异：**

| 维度 | 西方CEO | 中国CEO |
|------|---------|---------|
| 提问方式 | 直接提问OK | 必须用"请教"姿态 |
| 第一次见面 | 可以深入业务问题 | 以建立关系为主，轻触业务 |
| 挑战对方 | "Challenger Sale"有效 | 挑战=不给面子，关系终结 |
| 数据引用 | 引用公开财务数据OK | 需谨慎（暗示"我在调查你"） |
| 竞争对手 | 可以直接提 | 含蓄提（"行业同行"代替点名） |

**有效的中国式Discovery问法：**
1. "请教一下，贵司在XX领域的规划方向是什么？"（请教姿态）
2. "我们服务过的类似企业，普遍面临XX挑战，不知贵司的情况如何？"（用案例铺垫）
3. "最近政策在推XX方向，贵司这块的落地进度怎么样？"（借政策开口）
4. "行业里XX标杆做得比较好，不知您怎么看？"（用同行打开话题）
5. "张总在XX峰会上提到的那个方向，目前是什么阶段？"（show homework）

**绝对不能问的：**
- "您觉得贵司最大的问题是什么？"（暗示有问题=不给面子）
- "为什么还没有上XX系统？"（暗示落后）
- "您的预算是多少？"（第一次见面问=太商业化）
- "之前那个项目为什么失败了？"（揭短）

## 十二、中国CEO关系地图与权力动态

### 央企权力结构

```
党委书记/董事长（一把手，战略+政治）
    ↕ [实线权力关系]
总经理/党委副书记（二把手，运营+执行）
    ↕
副总经理/总会计师/总工程师（分管业务线）
    ↕
部门负责人（处长/部长级）
```

**关键动态：**
- 党委书记=董事长（通常同一人）= 真正的决策者
- 总经理虽管运营，但重大决策需党委书记同意
- 纪委书记不管业务，但有调查权 → 所有人忌惮
- **Sales implication:** 必须同时搞定"一把手"和"分管副总"。只搞定分管的，一把手可以一票否决；只搞定一把手，分管的可以消极执行。

### 大型民企权力结构

```
创始人/实际控制人（终极决策权）
    ↕
创业元老圈（影响力远超其title）
    ↕
职业经理人高管层（CEO/CFO/CTO等）
    ↕
中层管理
```

**关键动态：**
- 创始人的权力远远超过其公开title
- "开封元老"（创业团队）往往有特殊影响力和否决权
- 职业经理人即使title是CEO，在战略决策上仍需创始人点头
- 空降高管头1-2年是"试用期"，没有真正的自主决策权
- **Sales implication:** 找到创始人的"信任圈"里的人做引荐，比任何销售技巧都管用。

### 外企在华权力结构

```
全球/亚太总部
    ↕ [虚线汇报，实际决策权有限]
中国区总裁/CEO
    ↕ [实线管理]
各职能VP/总监
```

**关键动态：**
- 中国区总裁的自主权因公司而异（"strong China" vs "weak China"模式）
- 重大采购权限通常$100K-$500K，超出需全球/亚太审批
- 2024趋势：61%欧企已将IT采购权本地化（5年前仅30%）
- 全球采购框架vs本地best-of-breed存在持续张力
- **Sales implication:** 先搞定中国区（他们有动力证明"本地创新"），再通过他们影响全球决策。

### 政府关系在各类型中的角色

| 企业类型 | 政府关系重要度 | 核心连接对象 | 用途 |
|----------|--------------|-------------|------|
| 央企 | ★★★★★ | 国资委、主管部委 | 考核、人事、资源 |
| 地方国企 | ★★★★★ | 地方政府、省国资委 | 考核、土地、政策 |
| 大型民企 | ★★★★☆ | 发改委、工信部、地方政府 | 补贴、牌照、政策保护 |
| 专精特新 | ★★★★☆ | 工信部、地方科技局 | 认定、补贴、政府订单 |
| 外企在华 | ★★★☆☆ | 商务部、地方招商局 | 市场准入、合规协调 |
| 出海企业 | ★★★☆☆ | 商务部、目的地使馆 | 出海政策、贸易摩擦协调 |

## 十三、Do's & Don'ts 中国特别版

### ✅ DO（中国通用）

- **先关系，后业务。** 第一次见面的目标是"让对方愿意见你第二次"，不是推方案。
- **用中间人引荐。** 冷拓是最低效的方式。找到共同认识的人、行业协会、或对方信任的顾问做引荐。
- **准备标杆案例。** 中国客户最看重"同行谁在用"。案例越接近（同行业、同规模、同地区），说服力越强。
- **尊重层级。** 如果对方是副总裁，你的对接人至少应该是总监级。层级不对等=不被重视。
- **学会"表态"。** 在关键时刻明确表达支持对方的立场。模糊的中立在中国商业文化里=不可信任。
- **配合政策叙事。** 任何方案都要能对齐当前政策热词（新质生产力、数字化转型、信创等）。
- **patience。** 中国大客户销售周期天然比西方长，急于求成会被视为"不懂行"。

### ❌ DON'T（中国通用）

- **不要冷启动hard sell。** 没有关系基础直接推销=垃圾邮件。
- **不要在公开场合让对方难堪。** 面子比交易重要。当面质疑对方的决策=关系终结。
- **不要跳级。** 绕过对方的直接下属直接找老板=得罪整条线上的所有人。
- **不要在饭局上谈价格。** 饭局是建立信任的场景，不是商务谈判场景。
- **不要假设CXO title = 决策权。** 在很多中国企业，title不等于实权。先搞清楚实际决策链。
- **不要忽视"技术守门人"。** 在国企，处长/科长级虽没决策权，但有一票否决的"搅局"能力。
- **不要用公开财务数据质疑对方。** 这在中国文化里等于"我在调查你"。
- **不要催促。** "这个月底能签吗？"式的追问=low-class sales，尤其对高层。

### 企业类型特别 Do's & Don'ts

| 企业类型 | DO | DON'T |
|----------|-----|--------|
| **央企** | 对齐政策叙事（新质生产力）；走正式流程（招投标）；准备信创方案 | 试图"走关系"绕过流程（反腐高压）；忽视党委审批环节 |
| **地方国企** | 找地方政府关系做背书；配合地方产业政策 | 只盯总部不管分支机构（决策在地方） |
| **大型民企** | 找到创始人信任圈的人引荐；展示技术硬实力；快速响应 | 走太慢的流程（创始人会失去耐心）；PPT太多实操太少 |
| **专精特新** | 直接约创始人；展示如何帮助他们技术突破；提供灵活合作模式 | 用大企业那套流程来对接（他们没有采购部门）；价格太高 |
| **外企在华** | 准备英文材料+全球合规证明；对接本地IT负责人 | 忽视全球总部审批流程；只做中文沟通 |
| **出海企业** | 展示多国合规能力；提供多语言/多区域方案 | 只谈中国市场方案（他们需要全球视角） |
