## Asset 详细要点

### 1. Financial Market Data 数据资产（互联网券商）

- **目标客户**：互联网券商（OB）+ QHF；联系人：行情负责人、机构业务负责人
- **行业趋势**：交易数据年增长 20-40%，传统本地基础设施成本压力大；OB 从 C 端走向机构业务（IPO 分销、ESOP、定制化报告）作为第二增长曲线
- **核心痛点**：数据快速分发 / 监管安全 / 权限粒度（角色爆炸：1000 资源 × 100 主体 = 10 万策略） / 技术架构落后 / 数据质量时效
- **数据提供商生态**：
  - **实时数据**：Bloomberg、FactSet、ICE、LSEG、Nasdaq → AWS Private Link / Kinesis / MSK
  - **历史/另类**：Bloomberg、BMLL、FactSet、TICKDATA、Tradeweb → AWS Transfer Family / Data Exchange / S3 Access Points
- **核心架构**：S3 数据湖 + Lake Formation（治理 + 集中授权）+ Glue + 行级安全 RLS
- **合作伙伴**：Snowflake（76% 客户运行在 AWS）

### 2. Distributed Quantitative Investment（量化）

- **行业基线（2023）**：量化私募 1.63 万亿 + 量化公募 0.32 万亿
- **量化交易系统五要素**：数据 / Alpha 模型 / 风险模型 / 交易成本模型 / 投资组合模型
- **价值主张**：弹性扩展应对回测/信号生成/风险计算高负载，云端量化迁移
- **趋势**：国内监管收紧 → 私募转战海外，全球化投资组合策略

### 3. Investment Advisory（多 Agent 投顾）

- **创新点**：多 Agent 协同架构（PortfolioManagerAgent 主控 + 8 个领域专家 Agent + MCP 能力服务层）
- **三层记忆体系**：S3（持仓/历史）+ Bedrock Knowledge Base（产品说明/研报）+ OpenSearch Serverless（向量）
- **MCP 服务化**：定量分析 Lambda（回测/因子/风险）+ 配置计算（组合优化）+ 外部数据接口
- **三大效果**：输出稳定（确定性服务）、可审计、可对接生产
- **典型场景**：外汇期权交易辅助（Black-Scholes/Garman-Kohlhagen 定价 + Delta/Gamma/Vega/Theta/Rho + FpML 合约）

### 4. KYC Agent（Multi-Agent CDD）

- **场景**：数字银行开户、支付注册、券商开户
- **核心能力**：
  - 证件 OCR + 结构化提取
  - 证件真实性检测（防篡改/版式异常/有效期）
  - **Amazon Rekognition Face Liveness**：FaceMovementAndLightChallenge 主动式挑战，活体置信度三级风控（≥80% 通过 / 70-80% 复核 / <70% 拒绝）
  - 银行流水识别（PDF 多页拼接 + 跨页交易记录 + 智能分类）
- **业务效果**：开户从多次提交→一次完成，显著降低注册流失

### 5. Generative Data Insights / GenBI（金融 BI）

- **核心场景**：银行/资本市场/保险智能 BI 分析
- **典型分析**：CAGR 计算 / PE/PB 比率 / ROE 等盈利能力分析
- **三大痛点**：非技术人员门槛高 / 数据访问民主化不足 / 报表开发周期数天-数周
- **关键能力**：
  - **Text-to-SQL** + **OpenSearch RAG** + 实体识别（NER）
  - **数据 Profile 隔离** + **行级安全 RLS** + 数据防泄漏（仅返回 SQL 不返回数据）
  - Agent 任务拆分 + 链式推理（CoT）归因分析 + 自动可视化
- **支持模型**：Bedrock（Claude、Nova）+ SageMaker 自定义（DeepSeek 等）
- **部署**：CDK 一键到 AWS（ECS / OpenSearch / DynamoDB）
- **目标客户**：支付公司、银行、互联网券商、保险、互联网小贷

### 6. SMS Debt Risk Detection（短信共债识别）

- **场景**：贷前审批 + 贷后管理
- **痛点**：传统征信覆盖不足/滞后 / NLP 误判营销为授信 / 缺实时风险感知
- **能力**：
  1. 短信智能分类与结构化提取（Bedrock LLM 多级分类）
  2. 共债风险自动识别（多平台贷款交叉分析）
  3. 三级预警（高/中/软提醒）
  4. 可配置策略版本化（Prompt/规则/词库/模型/判例 → S3 版本化）
- **架构**：Bedrock + Step Functions（Map State 并行：消息分析 8 路 / 共债分析 6 路）+ Throttling 自动指数退避

### 7. GNN 实时反欺诈

- **方案**：Deep Graph Library（DGL）+ Amazon Neptune（图数据库）+ SageMaker
- **数据**：IEEE-CIS 在线金融交易，转化为异构图
- **完整流程**：数据注入 → 数据清洗 → 模型训练 → 模型更新上线
- **GitHub 实现**：CloudFormation 模板 + 自动化部署

### 8. FSI Connect 联络中心（金融行业最佳实践）

- **行业趋势**：客群细分化 / 渠道场景化 / 产品丰富化 / 体验极致化
- **传统客服痛点**：多渠道体验不一致 / 一次性问题解决率低 / 不同客群满意度差异 / 响应慢 / 多语言弱 / 构建复杂高成本
- **海外建设技术痛点**：跨区域延迟（东南亚/非洲）/ 数据合规（GDPR/中国本地化）/ 系统集成 / 多语言本地化 / 高成本
- **服务流程**：触达 → 服务 → 转化 → 分析

### 9. Post Call Analytics

- **能力**：全量通话质检 + 情感分析 + 实体抽取 + 合规检测
- **构建端到端服务质量与风控闭环**

### 10. 跨区域失效转移与自动恢复

- 金融合规对 RTO/RPO 严格要求
- 跨区域多活 + 自动失效转移与恢复

## 涉及 AWS 服务

- **AI/ML**：Bedrock（Claude、Nova）、SageMaker（DGL/GNN）、Rekognition Face Liveness、Comprehend、Transcribe、Lex
- **数据**：S3（数据湖）、Lake Formation、Glue、Redshift、RDS、OpenSearch、DynamoDB、Athena、Kinesis、MSK（Kafka）、Data Exchange、Transfer Family

- **基础设施**：ECS、EKS、Lambda、CDK、CloudFormation、Private Link
- **Agent**：Bedrock AgentCore Gateway + Strands Agents + MCP

- **量化私募/公募**：千象/明汯/九坤/幻方
