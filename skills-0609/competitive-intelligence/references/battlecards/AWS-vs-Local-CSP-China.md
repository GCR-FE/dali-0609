# AWS 中国地域 vs 本土友商（阿里 / 华为 / 腾讯）— 综合 Battlecard

> ⚠️ CI 内部使用条款：仅限内部使用，对比数据仅限 1对1 口头引用。

---

## TL;DR — AWS 中国 4 大独特竞争优势

| 维度 | AWS 优势 | 友商问题 |
|---|---|---|
| **1. 韧性架构** | Region/AZ 双隔离 + 单元化 + 控制/数据面分离 + 静态稳定性 | 阿里云 2022-24 三起 LSE，单 AZ 故障搞瘫整个 region |
| **2. 数据隐私** | Job Zero 安全；不和客户业务竞争 | 阿里集团/腾讯集团在零售/金融/物流等行业**与客户同业竞争** |
| **3. 网络能力** | 自建 400 GbE 全球骨干网 + 全 mesh AZ 互联 + Global Accelerator | 友商海外靠租用第三方运营商 |
| **4. 智能创新** | Bedrock + SageMaker + Trainium/Inferentia + 全球生态 | 华为 Bedrock 无对应；昇腾受 SMIC 工艺限制；OpenAI 仅企业准入 |

---

## 1. 韧性架构 — AWS 业务连续性的硬实力

### 1.1 业务价值（要先讲钱）

> **BCG 研究**：提高系统韧性可增加 **12-20% 利润**

业务中断的成本：
- 消费者期望"永远在线、永远可用"
- 业务中断时客户**有动机转向竞争对手**
- 严格行业法规要求应用比以往更具韧性
- Uptime Institute 2022 数据：**停机每小时成本 > $300,000**

### 1.2 AWS 韧性架构 4 层设计

| 层级 | 设计 | 说明 |
|---|---|---|
| **1. Region/AZ 模型** | Gartner 推荐方法 | 每 AZ 多个独立数据中心，水电网络独立；100km 内、metro fiber 完全冗余、加密 |
| **2. AZ 互联** | **全网状（FULL MESH）** | 单线路故障**秒级**路由收敛 |
| **3. 单元架构（Cellular）** | 客户无感知部署 | **控制故障爆炸半径**；配合灰度/回退发布机制 |
| **4. 控制面/数据面分离** | 静态稳定性设计 | EC2 启动后即使控制面挂了仍正常运行；IAM 控制面在 us-east-1，数据面分布每个 region |

### 1.3 友商问题 — 阿里云 LSE 案例

| 时间 | 故障 | 范围 | 时长 |
|---|---|---|---|
| 2022-12-18 | 香港 Region | 整个 region | **>12 小时** |
| 2023-11-12 | 控制台/API | 全球大部分核心服务 | 约 3h20m |
| 2024-07-02 | 上海 AZ N | 该 AZ 全部服务 | 38m |

## 2. 数据隐私与利益冲突

### 2.1 AWS 数据隐私四大承诺

1. **客户控制自己数据**（权限 / 区域 / 加密）；AWS 不未经同意访问、使用或共享
2. **支持最高隐私标准与合规认证**
3. **提供隐私控制工具**（高级访问、加密、日志）+ 一致可扩展的数据治理
4. **合同条款简单透明**

### 2.2 友商利益冲突

| 友商 | 集团业务 | 与之冲突的客户 |
|---|---|---|
| **阿里巴巴** | 淘宝/天猫/盒马/饿了么/菜鸟/优酷/蚂蚁 | 零售/物流/内容/金融 |
| **腾讯** | 微信/QQ/游戏/视频号/腾讯金融 | 互娱/社交/游戏/支付 |
| **华为** | 终端/汽车/能源/电信 | 设备厂商/车企/能源 |

> **销售话术**：让 EB 自己回答"你们行业有没有跟这家集团业务正面冲突的子公司？"

## 3. 销售战术 — 应对友商常见说辞

### 3.1 友商说"价格便宜 30%"
**回应**：
- 反问"对应哪个 SLA / 冗余级别 / CPU 代次"
- 阿里云 U 系列 / RDS GP 实例 = 减配版
- 把**故障损失**纳入 TCO（每小时 $300K+）

### 3.2 友商说"我们有客户案例"
**回应**：
- 案例 ≠ 当前客户业务匹配度
- 用 Frost & Sullivan《What Determines Cloud Resilience》报告反驳
- AWS 全球客户案例库 + 行业 reference 更深

### 3.3 友商说"我们能更快响应客户化需求"
**回应**：
- 警惕"私有 fork"风险——离开后无法迁移
- AWS 全球产品成熟度 + GA 服务 SLA 承诺更稳定
- 强调"标准化是出海的前提"
