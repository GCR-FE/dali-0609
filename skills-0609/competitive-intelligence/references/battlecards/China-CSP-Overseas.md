# 中国云厂商海外区域 vs AWS — Battlecard

> ⚠️ CI 内部使用条款。数据截止 **2025-12-01**，对比 GCR 以外（中国大陆/港/台外）地域。

## TL;DR — 中国客户出海四大选 AWS 理由

| 维度 | AWS | 阿里云 / 华为云 / 腾讯云（海外） |
|---|---|---|
| **覆盖广度** | **34 个 GCR 外区域 / 108 AZ**，每 region ≥3 AZ | 总数约 AWS 的 **1/3 ~ 1/2**，常无 3 AZ |
| **基础设施一致性** | 自建 + 自营骨干 + 透明披露 | 部分租运营商/合作伙伴运营，标准不统一 |
| **网络** | **400 GbE** 自建骨干，多 Tb 容量 | 海外多用第三方电信，未公开披露 |
| **ESG** | **2023 年 100% 绿电达成**（早 7 年） | 友商承诺 2030 |

## 关键数据点

### 区域和可用区
- AWS：全球 **38 region / 120 AZ**，GCR 外 34 region / 108 AZ
- 友商海外：约为 AWS 的 1/3 - 1/2，常缺 3+ AZ 配置
- AWS 计划新增 4 个区域；友商有**关停记录**：
  - 阿里云：悉尼、孟买
  - 腾讯云：孟买
  > 出海客户业务连续性的潜在风险

### 物理隔离与建设标准
- AWS：每 AZ 物理隔离 ≤100km，多个独立 DC，同一标准
- 友商：常租用本地运营商 DC，**部分区域由合作伙伴运营**：
  - 华为云欧洲、中东部分区域
  - 阿里云利雅得区域

### 宕机透明度
- AWS：**公开披露**全量宕机，海外云厂商中宕机时长和影响最低
- 友商：颗粒度粗，仅披露重大舆情事件，难评估稳定性

### 网络骨干
- AWS：**自建 400 GbE** 多 Tb 骨干网，确保低时延高可靠
- 友商：未透明披露，**很可能租第三方网络**（例：阿里云海外 DC 通过多家电信运营商接入）

### ESG 合规
- AWS：**2023 年达成 100% 可再生能源**（提前 7 年）；全球最大企业绿电采购商
- 阿里云、腾讯云：**目标 2030**
- AWS 参与创立《气候承诺》：2040 净零，2030 50% 碳中和

### 服务覆盖广度
- AWS：在 GCR 以外尤其欧美**最全面**，**最先**推出新服务
- 友商：海外服务可用性**有限**，新服务部署滞后
- 例：阿里云**倚天实例**（对标 Graviton）仅在 **2 个** GCR 外地域可用

## 销售战术 — 中国出海客户场景

### 异议 1："阿里云在东南亚市场份额高、价格便宜"
**回应**：
- 关停历史（悉尼/孟买）→ **业务连续性风险**
- 多 AZ 缺失 → 一个 AZ 故障 = region 故障
- 出海重点不是省钱而是**稳定/合规**——讲 12-20% 韧性利润提升

### 异议 2："华为云在中东、非洲布局领先"
**回应**：
- 部分区域**由合作伙伴运营**——服务标准不统一、SLA 责任不清
- AWS 中东区域是**自营**+ 跨区域骨干打通
- 强调中国合规（CIDPL、个保法）+ 出海合规（GDPR、CCPA）双合规框架

### 异议 3："腾讯云在游戏出海有优势"
**回应**：
- 游戏：AWS GameLift / GameTech 全球节点更密
- 关停孟买 → 印度市场用 AWS 孟买/海得拉巴更稳
- 全球加速器（Global Accelerator）+ CloudFront 边缘节点数量级领先
